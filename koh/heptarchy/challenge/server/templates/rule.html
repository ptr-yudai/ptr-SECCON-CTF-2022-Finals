What is this challenge?

How quickly and accurately can you reverse engineer programs?
There are some useful tools for decompiling a binary, such as IDA and Ghidra. However, none of them can recover the original source code 100%.
It is always human skills* that can decompile a semantically meaningful code.
Show your reverse engineering skills.

* Sometimes ChatGPT in these days :)

The Heptarchy

The Heptarchy is the seven petty programs that you'll conquer.
So, you need to reverse engineer the following programs as quick and accurate as possible:
  • C
  • C++
  • Rust
  • Go
  • D
  • Python
  • WebAssembly

Scoring System

The following figure shows the overview of this challenge:

For each hour, you can download a program binary named mission-X (X is from 1 to 7). Each binary is compiled using a common language. (See "The Heptarchy" for the list of languages.) Your task is to reverse engineer the binary and write the source code in that language. In total, you will have to analyze 7 different programs compiled from 7 different source codes.
Once you have decompiled the code, you must upload it to the server from this dashboard.
The server tests the uploaded code every 5 minutes (1 round). Your code is compiled into a binary, and the server calculates how similar the produced binary is to the distributed mission binary. The smaller the difference, the higher your rank. The team with the smallest difference gets 20 points. The team with the second highest rank gets 18 points, the third 16 points, and so on.
If multiple teams upload code with the same diff score, they will receive the same score. For example, if two teams record a diff of 0 (the highest score), they will both receive 20 points.
In any of the following cases, the team will not receive any points for the round:

  • The team does not upload any code for the language. (You don't have to upload the same code every round.)
  • The uploaded code does not compile within 10 seconds.
  • The diff calculation does not finish in 10 seconds, meaning there is too much difference.
The algorithm for calculating the diff is based on this paper.

What you MUST NOT do

We may deduct points or disqualify your team if we notice any of the following violation:

  • Attempt to attack this dashboard
  • Attempt to overwrite other teams' source codes by any means
  • Attempt to exploit the scoring system (the comparator)
  • Attempt to send large requests to this dashboard at once
  • Overloading CPU or consuming a large amount of disk space

Kindly tell the admin if you find any suspicious behavior or bugs :)
